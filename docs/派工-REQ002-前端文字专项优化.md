# 派工 · REQ-002 · 前端文字专项优化（不出戏）

> **状态**: ❌ 完全未实现
> **提交时间**: 2026-05-28
> **目标**: 桌宠核心在于"不出戏"。所有用户可感知的文字统一人设语气，消除机械感。
>
> **审查方式**: Agent Team 双路并行审查（前端 8 文件 + 后端 10 文件），逐行扫描所有用户可见字符串。
> **审查结果**: 共发现 **116 处**出戏问题（前端 81 + 后端 35）。
>
> **人设参考**: "清和" — 清冷沉静、温和专注、知性优雅、偶尔小幽默。
> **禁止**: 系统感、营销感、机器人感、少女系用语（波浪号~、"呀"、"嘿嘿"等）。

## 四、修改原则

### 4.1 语言风格规则

| 规则 | 示例 |
|------|------|
| 不用波浪号 `~` | `来了~` → `来了，` |
| 不用"呀"、"嘿嘿"、"啦"等少女系语气词 | `搞定啦` → `做好了` |
| 不用"！"感叹号（除非 LLM 主动输出） | `你好呀！` → `你好，` |
| 冒号格式 `：` 改为破折号 `—` | `结果：xxx` → `结果 — xxx` |
| 技术术语软化 | `LLM 配置` → `语言模型` |
| 过于正式的动词改口语化 | `启用` → `打开`，`检测` → `看看` |
| 状态标签去系统感 | `已完成` → `搞定了`，`待审` → `等你看看` |
| 空状态文案不要模板化 | `有新任务时会出现在这里` → `有新安排了会在这儿出现` |

### 4.2 保持不变的文案

以下文案已经符合人设或属于通用 UI 操作词，不需要修改：

- `和清和聊聊`、`还没有对话`、`想聊什么都可以`（ChatPanel）
- `说点什么...`、`发送`、`关闭`（通用操作）
- `保存`、`取消`、`确定`（按钮文字）
- `隐藏`、`退出`（菜单项）
- tray.rs 全部菜单文字
- initiative.rs 全部主动触发语
- 情绪名称：`开心`、`平静`、`无聊`、`害羞`、`低落`、`懊恼`
