## 1. 项目定位

Personal Conductor 是一个桌面 AI Agent 运行时，核心能力是：

- **Goal 驱动**：用户提出目标 → OODA 循环自动推进 → 拆分为 AgentTask → 多 Agent 并行执行 → 自动验收
- **混合模型指派**：`ModelResolver` 根据 caller context / OODA phase / hint / policy 动态路由到不同 LLM
- **多 Agent 编排**：`AgentTeam` + `AgentTeamMember` 管理执行团队，6 态生命周期
- **桌面壳**：Tauri + React，IPC 事件驱动 UI 更新

## 8. 架构决策记录

| 决策 | 选择 | 理由 |
|---|---|---|
| 事件总线 | 双总线并行（audit + chat_turn_events） | audit 是全量日志，turn_events 是 per-request timeline；短期不统一 |
| Member 状态驱动 | worker 层直接调 `set_member_status` | 不引入新 trait，最小改动；P1 再抽 `GoalTaskExecutor` |
| MCP Router | 保留静态路由 + `model.invoke` 返回不可用 | 短期不删，P2 再决定接真实治理还是删 |
| backend_id | `{backend_kind}:{profile_id}` | 有 profile_id 时可区分同 kind 不同配置 |
| 前端刷新 | Tauri 事件 + HTTP 轮询混合 | 关键状态用 Tauri 事件即时刷新，详情面板用轮询 |
