## 二、设计目标

```
当前：工具执行 → 原始 JSON → 用户看到技术输出
目标：工具执行 → 角色化表达 → 清和用性格化语言反馈 → 情绪反应 + 亲密度变化
```

**三层增强**：

1. **表达层**（Expression Engine）：工具结果 → 角色化文本 + 表情切换
2. **记忆层**（Memory Integration）：语义记忆 → 注入 system prompt → 上下文感知对话
3. **关系层**（Affection System）：交互质量 → 亲密度累积 → 关系阶段 → 性格微调

```rust
/// 情绪维度（PAD 模型简化版）
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Mood {
    /// 效价：-1.0（消极）~ 1.0（积极）
    pub valence: f32,
    /// 唤醒度：0.0（平静）~ 1.0（兴奋）
    pub arousal: f32,
    /// 时间戳
    pub updated_at: chrono::DateTime<chrono::Utc>,
}

impl Default for Mood {
    fn default() -> Self {
        Self { valence: 0.3, arousal: 0.2, updated_at: chrono::Utc::now() }
    }
}

/// 关系阶段（由亲密度驱动）
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum RelationshipStage {
    Stranger,   // 0-19
    Acquaintance, // 20-39
    Colleague,  // 40-59
    Friend,     // 60-79
    CloseFriend, // 80-100
}

impl RelationshipStage {
    pub fn from_value(value: u32) -> Self {
        match value {
            0..=19 => Self::Stranger,
            20..=39 => Self::Acquaintance,
            40..=59 => Self::Colleague,
            60..=79 => Self::Friend,
            _ => Self::CloseFriend,
        }
    }

    /// 关系阶段影响称呼和语气
    pub fn tone_hint(&self) -> &'static str {
        match self {
            Self::Stranger => "礼貌正式",
            Self::Acquaintance => "友善但保持距离",
            Self::Colleague => "专业配合",
            Self::Friend => "轻松随意",
            Self::CloseFriend => "亲密关心",
        }
    }
}

/// 空闲阶段（影响主动行为策略）
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum IdlePhase {
    /// 刚空闲（< 5min），不打扰
    JustIdle,
    /// 短暂空闲（5-15min），偶尔关注
    ShortIdle,
    /// 长时间空闲（15-30min），考虑问候
    ModerateIdle,
    /// 超长空闲（> 30min），主动问候
    LongIdle,
}

/// 组合表达状态
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ExpressionState {
    pub mood: Mood,
    pub relationship: RelationshipStage,
    pub idle_phase: IdlePhase,
    pub last_tool_result: Option<ToolResultSentiment>,
}
```

### 3.2 工具结果 → 角色化文本

```rust
/// 将工具执行结果转换为清和风格的反馈文本
pub fn format_tool_result_for_pet(
    tool_name: &str,
    result: &ToolExecutionResult,
    state: &ExpressionState,
) -> String {
    let sentiment = classify_result(result);
    let stage_tone = state.relationship.tone_hint();

    match &sentiment {
        ToolResultSentiment::Success { summary } => {
            // 根据关系阶段调整语气
            match state.relationship {
                RelationshipStage::Stranger | RelationshipStage::Acquaintance => {
                    format!("已完成{}。{}", tool_name, summary)
                }
                RelationshipStage::Colleague => {
                    format!("{}搞定了！{}", tool_name, summary)
                }
                RelationshipStage::Friend | RelationshipStage::CloseFriend => {
                    // 亲密关系下可以更活泼
                    let prefix = if state.mood.valence > 0.5 { "嘿嘿，" } else { "嗯，" };
                    format!("{}{}——{}", prefix, tool_name, summary)
                }
            }
        }
        ToolResultSentiment::Failure { error } => {
            match state.relationship {
                RelationshipStage::Stranger => {
                    format!("抱歉，{}未能完成。{}", tool_name, truncate(error, 100))
                }
                _ => {
                    format!("呀，{}出了点问题呢……{}", tool_name, truncate(error, 100))
                }
            }
        }
        ToolResultSentiment::PartialSuccess { summary, warnings } => {
            format!("{}基本完成了，{}。不过要注意：{}", tool_name, summary, warnings.join("；"))
        }
    }
}
```

### 3.3 情绪更新规则

```rust
impl Mood {
    /// 工具成功 → 效价 +0.1, 唤醒 +0.05
    pub fn on_tool_success(&mut self) {
        self.valence = (self.valence + 0.1).min(1.0);
        self.arousal = (self.arousal + 0.05).min(1.0);
        self.updated_at = chrono::Utc::now();
    }

    /// 工具失败 → 效价 -0.15, 唤醒 +0.1（挫败感）
    pub fn on_tool_failure(&mut self) {
        self.valence = (self.valence - 0.15).max(-1.0);
        self.arousal = (self.arousal + 0.1).min(1.0);
        self.updated_at = chrono::Utc::now();
    }

    /// 长时间无交互 → 情绪回归中性
    pub fn decay(&mut self) {
        let elapsed = chrono::Utc::now() - self.updated_at;
        if elapsed > chrono::Duration::minutes(30) {
            // 衰减到中性
            self.valence += (0.3 - self.valence) * 0.1;
            self.arousal *= 0.9;
            self.updated_at = chrono::Utc::now();
        }
    }
}
```

## 四、模块二：记忆-人格整合（Memory-Personality Integration）

### 4.1 Embedding 引擎替换

**当前**：`generate_embedding()` 使用 hash PRNG 生成 384 维伪向量。

**方案**：引入 `fastembed-rs` 本地 embedding。

```toml
# crates/conductor-core/Cargo.toml
[dependencies]
fastembed = "4"
```

```rust
// memory.rs 中替换 generate_embedding()
use fastembed::{EmbeddingModel, InitOptions, TextEmbedding};

static EMBEDDER: std::sync::OnceLock<TextEmbedding> = std::sync::OnceLock::new();

fn get_embedder() -> &'static TextEmbedding {
    EMBEDDER.get_or_init(|| {
        TextEmbedding::try_new(InitOptions {
            model_name: EmbeddingModel::BGESmallENV15,
            show_download_progress: true,
            ..Default::default()
        }).expect("failed to initialize embedding model")
    })
}

pub fn generate_embedding(text: &str) -> Vec<f32> {
    let model = get_embedder();
    model.embed(vec![text])
        .expect("embedding failed")
        .into_iter()
        .next()
        .unwrap()
}
```

**模型选择**：`BAAI/bge-small-en-v1.5`
- 维度：384（与现有表结构兼容）
- 大小：~50MB（首次下载后缓存）
- 速度：~2ms/句（CPU）
- 质量：MTEB 排名小型模型前列

**迁移注意**：
- 现有 dummy embedding 数据需清除：`DELETE FROM memory_embeddings;`
- `MemoryChunk` 表的 `embedding_model` 字段标记为 `"bge-small-en-v1.5"`

### 4.2 记忆注入 System Prompt

在 `chat.rs` 的 `build_system_prompt()` 中追加记忆上下文：

```rust
async fn build_system_prompt(/* existing params */) -> String {
    let mut prompt = existing_prompt;  // 现有 persona + skills + task context

    // 追加相关记忆
    if let Ok(memories) = memory::search_memory(&user_message, 5).await {
        if !memories.is_empty() {
            prompt.push_str("\n\n## 相关记忆\n");
            for mem in &memories {
                prompt.push_str(&format!(
                    "- [{}] {} (相关度 {:.2})\n",
                    mem.category, mem.content, mem.relevance_score
                ));
            }
        }
    }

    // 追加近期对话摘要
    if let Ok(summaries) = memory::get_recent_summaries(3).await {
        if !summaries.is_empty() {
            prompt.push_str("\n\n## 近期对话摘要\n");
            for s in &summaries {
                prompt.push_str(&format!("- {}\n", s.summary));
            }
        }
    }

    prompt
}
```

**调用时机**：每次 `llm_answer()` 时构建，利用 prompt cache（5min TTL）减少重复计算。

### 4.3 自动记忆生命周期

```rust
/// 对话结束后自动摘要存储
pub async fn auto_summarize_conversation(
    messages: &[ChatMessage],
) -> Result<()> {
    if messages.len() < 4 {
        return Ok(());  // 太短不值得摘要
    }

    let llm_config = config::load().await?.llm;
    let request_config = LlmRequestConfig::from_config(&llm_config);

    let summary_prompt = "请用一句话总结以下对话的关键信息（不超过50字）：";
    let conversation = messages.iter()
        .map(|m| format!("{}: {}", m.role, m.content))
        .collect::<Vec<_>>()
        .join("\n");

    let summary = llm::call(
        &llm_config.model,
        summary_prompt,
        &conversation,
        &request_config,
    ).await?;

    // 存为记忆 chunk
    let chunk = MemoryChunk {
        id: uuid::Uuid::new_v4().to_string(),
        content: summary.clone(),
        category: "conversation_summary".to_string(),
        source: "auto_summary".to_string(),
        importance: 0.5,
        created_at: chrono::Utc::now(),
        accessed_at: chrono::Utc::now(),
        access_count: 1,
    };
    memory::store_chunk(chunk).await?;

    // 生成 embedding
    let embedding = memory::generate_embedding(&summary);
    memory::store_embedding(/* chunk_id, embedding */).await?;

    Ok(())
}
```

**触发条件**：会话结束（用户关闭面板 / 超过 10min 无新消息）。

### 5.2 交互影响规则

```rust
/// 交互类型对亲密度的影响
pub enum InteractionType {
    /// 正常对话
    Chat { quality: f32 },  // 0.0-1.0，由对话长度/用户反馈估算
    /// 工具执行成功
    ToolSuccess,
    /// 工具执行失败
    ToolFailure,
    /// 用户主动打开面板
    UserInitiated,
    /// 主动问候被回应
    ProactiveGreeted,
    /// 主动问候被忽略
    ProactiveIgnored,
    /// 连续登录（每日首次交互）
    DailyReturn,
}

impl AffectionState {
    pub fn record(&mut self, interaction: InteractionType) {
        let delta = match interaction {
            InteractionType::Chat { quality } => (quality * 3.0) as i32,  // 0-3
            InteractionType::ToolSuccess => 1,
            InteractionType::ToolFailure => -1,
            InteractionType::UserInitiated => 2,
            InteractionType::ProactiveGreeted => 3,
            InteractionType::ProactiveIgnored => -2,
            InteractionType::DailyReturn => {
                // 连续登录奖励递增
                let bonus = self.consecutive_days.min(7) as i32;
                bonus
            }
        };

        self.value = (self.value as i32 + delta).clamp(0, 100) as u32;
        self.stage = RelationshipStage::from_value(self.value);
        self.last_interaction = chrono::Utc::now();
        self.interaction_count += 1;
    }
}
```

### 5.4 连续登录追踪

```rust
fn update_consecutive_days(state: &mut AffectionState) {
    let today = chrono::Local::now().date_naive();
    let last_date = state.last_interaction.with_timezone(&chrono::Local).date_naive();

    if last_date == today {
        return;  // 同一天，不更新
    } else if last_date + chrono::Days::new(1) == today {
        state.consecutive_days += 1;  // 连续
    } else {
        state.consecutive_days = 1;  // 断了，重新开始
    }
}
```

## 十、设计评审结论

**优势**：
- 完全复用现有基础设施（SQLite、ToolRegistry、ActivityVariant），不引入新架构
- 表达引擎是纯函数中间件，可选启用，不修改工具返回值
- 亲密度系统数据模型已有，只需填充逻辑
- fastembed 本地运行，无隐私泄露风险

**需注意**：
- persona.rs 与 config.persona 的统一是技术债，但不应阻塞 MVP
- embedding 替换后需全量重建索引，应提供 CLI 子命令
- 亲密度设计避免"刷分"：ProactiveIgnored 负反馈 + 每日上限

**可行性判定**：**GO**。MVP 8h 可完成核心闭环，其余增量迭代。
