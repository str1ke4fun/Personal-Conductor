# 桌宠静态 PNG 动态化与前端综合优化方案

> 范围：清和（Qinghe）桌宠前端；不再推进 Live2D，改用静态 PNG + CSS/SVG 复合动画撑起"活物感"。
> 目标：让用户感知到"她活了、有反馈、有性格"，且不引入新依赖、不破坏后端 LLM/Agent 链路。

## 0. 总体原则

1. **零新依赖**：纯 CSS + SVG + 少量 React 组件；不引 framer-motion / Lottie。
2. **所有动画 = 状态变量驱动**：用 CSS 自定义属性当总线，组件只声明"我在什么状态"，不写"我要做 1.02 倍缩放"这种细节。
3. **图越少越好**：每套主形象保留 5-7 张关键帧图即可，其余装饰全部 SVG/CSS。
4. **离线降级链**：CSS 动画 → SVG 装饰 → 静态 PNG → 内联剪影 SVG。任意一层坏掉不破窗。

## 2. PNG 动态化核心技术栈

### 2.1 三层架构

```
┌──────────────────────────────────┐  ← SVG 装饰层 (绝对定位, 可交互)
│ 光环 / 心情脸 / 状态徽章 / 气泡   │
├──────────────────────────────────┤
│        主形象 PNG (CSS 控制)      │  ← 主体层 (transform/filter 动画)
│                                  │
├──────────────────────────────────┤
│  接触阴影 + 地面光晕 (CSS 伪元素) │  ← 环境层 (filter 动画, 极轻)
└──────────────────────────────────┘
```

### 2.2 关键 CSS：让 transform 数字本身可插值

```css
@property --breath {
  syntax: '<number>';
  initial-value: 1;
  inherits: false;
}
@property --sway {
  syntax: '<angle>';
  initial-value: 0deg;
  inherits: false;
}

.pet-body {
  --breath: 1;
  --sway: 0deg;
  transform: scale(var(--breath)) rotate(var(--sway)) translateY(0);
  transform-origin: 50% 92%;       /* 脚底为支点，像真生物 */
  transition: transform 200ms ease;
  will-change: transform;
  animation: breath 7s ease-in-out infinite,
             sway 11s ease-in-out infinite;
}

@keyframes breath {
  0%, 100% { --breath: 1; }
  50%      { --breath: 1.025; }
}
@keyframes sway {
  0%, 100% { --sway: -1.2deg; }
  50%      { --sway: 1.2deg; }
}
```

`@property` 让 CSS 变量可以平滑插值。`transform-origin: 50% 92%` 是关键——从脚底呼吸，而不是像放大镜。

### 2.3 眨眼：纯 CSS mask 收缩

```css
.pet-blink {
  position: absolute;
  inset: 0;
  background: var(--pet-image) center/contain no-repeat;
  animation: blink 5.2s infinite;
}

@keyframes blink {
  0%, 92%, 100% { clip-path: inset(0 0 0 0); }
  94%           { clip-path: inset(var(--eye-y, 18%) 0 calc(100% - var(--eye-y, 18%) - var(--eye-h, 6%)) 0); }
  96%           { clip-path: inset(0 0 0 0); }
}
```

眼位 (`--eye-y`/`--eye-h`) 作为变量由 art 提供。

### 2.4 鼠标"侧头"：纯 transform

```css
.pet-body {
  --head-tilt: 0deg;
  transform: scale(var(--breath)) rotate(calc(var(--sway) + var(--head-tilt)));
  transition: transform 200ms ease, --head-tilt 180ms ease-out;
}
```

```tsx
// PetBodyShell.tsx
useEffect(() => {
  const onMove = (e: MouseEvent) => {
    const rect = el.getBoundingClientRect();
    const dx = (e.clientX - (rect.left + rect.width / 2)) / window.innerWidth;
    root.style.setProperty('--head-tilt', `${dx * 6}deg`);
  };
  window.addEventListener('mousemove', onMove, { passive: true });
  return () => window.removeEventListener('mousemove', onMove);
}, []);
```

### 2.5 一次性动作触发器

```css
.pet-body.is-bumped    { animation: bump 480ms cubic-bezier(.34,1.56,.64,1); }
.pet-body.is-shaking   { animation: shake 240ms linear 2; }
.pet-body.is-welcoming { animation: welcome 1.2s ease; }

@keyframes bump   { 0%{transform:scale(1)} 40%{transform:scale(1.08)} 100%{transform:scale(1)} }
@keyframes shake  { 0%,100%{transform:translateX(0)} 25%{transform:translateX(-3px)} 75%{transform:translateX(3px)} }
@keyframes welcome { 0%{transform:translateY(8px) scale(.96)} 40%{transform:translateY(-6px) scale(1.04)} 100%{transform:translateY(0) scale(1)} }
```

React 端通过 `key` 重新挂 class 强制重放。

### 6.2 拖动防误触

```tsx
const dragState = useRef({ startX: 0, startY: 0, moved: 0 });
async function onMouseDown(e: React.MouseEvent) {
  if (e.button !== 0) return;
  dragState.current = { startX: e.clientX, startY: e.clientY, moved: 0 };
}
async function onMouseMove(e: React.MouseEvent) {
  const dx = e.clientX - dragState.current.startX;
  const dy = e.clientY - dragState.current.startY;
  dragState.current.moved = Math.hypot(dx, dy);
  if (dragState.current.moved > 4) {
    // 触发 startDragging
  }
}
```

> 4px 阈值，避免"想点状态点但被解释为拖动"。

## 7. 气泡与对话系统

### 7.1 队列化

```ts
type Bubble = {
  id: string;
  kind: 'chat' | 'self' | 'notification' | 'system';
  content: string;
  durationMs?: number;
  onClick?: () => void;
  priority?: number;   // 0..100
};
```

- 默认每条 6s（鼠标 hover 时暂停计时）。
- 高优先级（proactive/error）插队。
- 同时只显示一条，维护一个最大 3 条的"待播"队列。

### 7.2 三种气泡视觉区分

| 气泡 | 位置 | 配色 | 装饰 |
|---|---|---|---|
| ChatBubble（用户问/助手答） | 底部偏上 | 白底 + 头像小圆 | 左/右上 8px 头像 |
| SelfBubble（桌宠主动开口） | 头顶下方 | 主色淡彩 + 0.6 透明 | "💭" 装饰条 |
| NotificationBubble（系统通知） | 底部 | urgency 颜色 | 图标 + "✕" 关闭 |

### 7.3 打字机感

```tsx
function TypingBubble({ text, speed = 24 }) {
  const [shown, setShown] = useState('');
  useEffect(() => {
    let i = 0;
    const t = setInterval(() => {
      i++;
      setShown(text.slice(0, i));
      if (i >= text.length) clearInterval(t);
    }, speed);
    return () => clearInterval(t);
  }, [text, speed]);
  return <span>{shown}<span className="cursor">▍</span></span>;
}
```

## 10. 事件流与状态机收口

### 10.1 单一状态源

```
Backend (Rust)
  ├─ avatar.rs        → emit 'pet_avatar_changed' (主形象锁/切换)
  ├─ affection.rs     → emit 'pet_expression'      (含 affection_value)
  ├─ expression.rs    → emit 'pet_expression'      (含 mood_zone/affection)
  ├─ initiative.rs    → emit 'pet_self_bubble'     (主动开口)
  ├─ smart_monitor.rs → emit 'pet_state'           (idle/working/update/quiet/new_task)
  └─ quiet_for_minutes→ emit 'pet_state'/'pet_trigger_anim'

Frontend
  └─ usePetVisualState 唯一聚合点
     ├─ usePetMood     (mood + 衰减)
     ├─ usePetAnimation(派生 CSS 变量 + class)
     └─ useAffectionGlow(关系 + 仪式触发)
```

### 10.2 修双消费

- `AvatarRenderer` 不再直接 listen，**只接收 visualState prop**。
- `MoodIndicator` / `AffectionBadge` 也改为读 hook，不直接 listen。
- 所有 event 入口唯一在 `usePetVisualState`。

### 10.3 资源 404 自检

启动后 5s 内 `Promise.all` 探测每张 PNG，失败统一进 self-bubble："换装路径坏了，去设置看看"。

### 10.4 修 fallback 链

```ts
type AssetState = 'mapped' | 'mood' | 'thinking' | 'neutral' | 'video' | 'inline-svg';
const [assetState, setAssetState] = useState<AssetState>('mapped');

onError = () => setAssetState(s => NEXT[s]);
const NEXT: Record<AssetState, AssetState> = {
  mapped: 'mood', mood: 'thinking', thinking: 'neutral',
  neutral: 'video', video: 'inline-svg', 'inline-svg': 'inline-svg',
};
```

`inline-svg` 是终极兜底：内置极简剪影 SVG，绝不黑屏。

## 11. 性能与降级

- **GPU 加速**：所有 transform 动画用 `will-change: transform`
- **prefers-reduced-motion**：用户系统层关动效时，禁掉 breath/sway/blink，只保留基础姿态
- **低帧率模式**：`@media (prefers-reduced-motion: reduce)` 同步降到 0.5 倍率
- **绘制节流**：`requestAnimationFrame` 合并 `mousemove`，头像侧头限制 30Hz

```css
@media (prefers-reduced-motion: reduce) {
  .pet-body, .pet-blink, .mood-face, .status-badge { animation: none !important; }
}
```
