# Steam 上架路线 - 暂不推进

> 状态: 暂不推进
> 最近评估: 2026-05-27
> 适用范围: 未来公开发行/商业化评估

## 结论

当前不把 Steam 上架纳入 v0.1.x 或近期打包目标。技术分发可行，但产品定位、隐私披露、第三方素材授权和审核准备成本都高于当前阶段收益。

短期优先级:

1. 先完成 Windows 内测版和正式安装包。
2. 先把首次启动配置、安装路径、自启动、WebView2、数据目录、代码签名等基础分发问题补齐。
3. Steam 仅保留为中长期候选渠道，不占用当前 P0/P1 工程排期。

## 暂不推进原因

### 产品定位尚未稳定

当前项目更像 AI 桌宠/个人工作助手，而不是标准游戏。若作为 Game 上架，需要补足真实游戏循环，例如养成、任务推进、收集、成长、反馈、成就或剧情事件。若作为 Software 上架，需要按工具类产品解释清楚用途、隐私边界和外部服务依赖。

### 审核材料成本较高

Steam 上架需要准备商店页、截图、视频、胶囊图、系统需求、定价、构建上传、测试分支、隐私说明和审核材料。当前版本仍处于内测打包阶段，先投入 Steam 材料会分散工程重点。

### 隐私与安全边界需要先产品化

项目会涉及 API key、LLM 请求、前台窗口感知、Claude/Codex hook、任务状态和聊天历史。上架前必须做到:

- 用户主动同意后才开启 hook 和前台窗口感知。
- 首次启动时明确说明哪些数据会本地保存、哪些内容会发送给 LLM 服务。
- API key 不明文裸放在易误传的 release 目录中。
- 卸载时区分删除程序和保留/清理用户数据。

### 第三方素材授权需要逐项确认

Live2D/Cubism/Hiyori、视频、图片、字体、音效等素材都要确认商业分发和 Steam 再分发权限。任何来源不清的素材都不能进入公开渠道。

## 未来重新评估触发条件

满足以下条件后再重新评估 Steam:

- Windows 正式安装包已经稳定，干净机器可安装、启动、卸载。
- 首次启动引导完成，用户可配置 LLM、隐私选项和 hook 开关。
- 产品定位确定为 Game 或 Software，且商店页叙述能真实匹配功能。
- 第三方素材授权清单完成。
- 至少有 1 个可演示的核心玩法或明确工具闭环。
- 已有可公开展示的截图、视频和图标资产。

## 如果未来走 Game 路线

需要先补以下产品工作:

1. 定义核心游戏循环: 工作任务 -> 桌宠反馈 -> 情绪/好感/成长 -> 解锁表达或事件。
2. 定义失败和长期目标: 连续专注、任务完成、陪伴等级、每日事件。
3. 定义可展示内容: 截图里必须能看出它是可玩的桌宠/养成体验，而不是纯工具面板。
4. 接入可选 Steam 功能: 成就、Steam Cloud、统计或社区物品可以后置，但至少要有明确使用理由。
5. 离线可启动: 即使没有 API key，也要能进入基础体验。

## 如果未来走 Software 路线

需要先补以下产品工作:

1. 明确类别: 更接近 productivity/player tool/desktop companion，而不是传统游戏。
2. 明确价值: 帮助用户管理任务、感知工作上下文、通过桌宠进行低干扰提醒。
3. 明确边界: 外部 LLM 由用户自行配置，默认不上传本地内容。
4. 明确隐私: 写出数据保存位置、网络请求目的、如何删除数据。
5. 明确付费模式: 一次性付费、免费加自备 API key，或后续订阅都需要单独评估。

## 技术路线备忘

Steam 不建议把普通安装包作为主程序交给用户二次安装。更合理的方式是上传已经解包的应用目录到 Steam depot，由 Steam 客户端负责安装和更新，并把 launch option 指向主程序 exe。

未来 Steam build 应包含:

- `conductor-desktop.exe`
- CLI sidecar `conductor.exe`
- Live2D/头像/前端资源
- 默认配置模板
- 隐私说明和许可证文件
- 必要运行时安装脚本或运行时检测逻辑

需要避免:

- 默认修改用户 `.claude` 或 `.codex` 配置。
- 默认扫描用户项目目录。
- 启动即要求 API key 才能进入主界面。
- 把调试日志、个人 API key、开发机绝对路径带入包内。
