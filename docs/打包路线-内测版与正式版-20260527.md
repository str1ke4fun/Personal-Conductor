# 打包路线 - 内测版与正式版

> 最近更新: 2026-05-27
> 目标平台: Windows 10/11 x64
> 当前结论: 内测优先便携 ZIP，正式版优先 NSIS 安装包

## 当前项目状态

项目已经具备 Windows 桌面端基础:

- 桌面壳: Tauri 2 + React/Vite。
- 主要产物: `target/release/conductor-desktop.exe`。
- 资源: Live2D、头像视频、前端静态资源。
- sidecar 候选: `conductor.exe` CLI，用于 Claude/Codex hook handler。

当前阻碍即开即用的关键问题:

- `tauri.conf.json` 中 `bundle.active` 仍为 `false`，还没有正式启用 Tauri bundler。
- `conductor-core::paths::root()` 默认回退到开发机路径 `I:\personal-agent`，用户机器上不可用。
- LLM API key、hook、窗口感知和数据目录还缺少首次启动引导。
- 正式安装包还缺少自定义安装路径和开机自启动选项。

## 分发目标

### A. 内测版: 便携 ZIP

用途: 小范围手动分发、快速验证功能，不追求完整安装体验。

交付物:

- `Personal-Conductor-v0.1.0-internal.zip`
- `bin/conductor-desktop.exe`
- `bin/conductor.exe`
- `resources/`
- `state-template/`
- `docs/`
- `README-内测启动.md`

验收标准:

- 解压到任意目录后可启动。
- 首次启动不会依赖 `I:\personal-agent`。
- 没有 API key 也能进入主界面，并提示配置 LLM。
- Live2D/头像资源可正常加载。
- 不默认启用 Claude/Codex hook。
- 不默认开启开机自启动。
- 日志、SQLite、配置写入用户数据目录或便携包内 `data/`，二者只能选一个明确策略。

推荐策略:

- 内测 ZIP 使用便携模式: 程序目录下 `data/` 保存状态，方便打包回收问题现场。
- 用户显式配置 LLM 后再写入配置。
- hook 开关放在设置页里，由用户手动启用。

### B. 正式版: NSIS 安装包

用途: 面向普通 Windows 用户，提供完整安装、卸载和配置体验。

交付物:

- `Personal-Conductor-Setup-v0.1.0-x64.exe`
- 可选: `Personal-Conductor-v0.1.0-portable.zip`
- `LICENSES/` 或 `THIRD_PARTY_NOTICES.md`
- `隐私说明.md`

最低验收标准:

- 安装向导允许用户自定义安装路径。
- 安装向导允许用户选择是否开机自启动，默认建议关闭。
- 安装完成后可选择立即启动。
- 程序数据写入 `%APPDATA%` 或 `%LOCALAPPDATA%` 下的稳定目录。
- 卸载程序可删除程序文件，并明确是否保留用户数据。
- WebView2 缺失时有明确处理方案。
- 安装包和主 exe 具备代码签名计划，正式公开分发前必须签名。

## 安装引导要求

正式版至少需要一个安装引导，覆盖两个层面。

### 1. 安装器引导

安装器必须提供:

- 安装路径选择页。
- 开机自启动复选框。
- 创建桌面快捷方式复选框。
- 创建开始菜单快捷方式。
- 安装完成后启动应用复选框。

建议默认值:

- 安装路径: `%LOCALAPPDATA%\Programs\Personal Conductor`，优先避免 UAC。
- 开机自启动: 关闭。
- 桌面快捷方式: 开启。
- 开始菜单快捷方式: 开启。
- 安装后启动: 开启。

### 2. 应用首次启动引导

首次启动必须提供:

- 选择数据目录策略: 默认用户数据目录，可选便携数据目录。
- 配置 LLM provider/base URL/model/API key。
- 隐私确认: 前台窗口感知、LLM 请求、聊天历史和本地存储说明。
- hook 配置: Claude/Codex 默认关闭，用户主动开启。
- 开机自启动二次确认: 设置页可随时开关。

## 工程任务拆分

### P0: 打包前必须完成

1. 移除开发机硬编码路径。
   - `CONDUCTOR_ROOT` 仍可作为覆盖项。
   - 默认 root 改为用户数据目录或便携目录。
   - 所有 SQLite、日志、配置、任务文件都从统一路径解析。

2. 增加首次启动初始化。
   - 如果没有配置文件，自动生成默认配置。
   - API key 为空时不报错退出。
   - 首次启动引导状态写入本地配置。

3. 启用 Tauri bundler。
   - `bundle.active = true`
   - Windows 正式目标优先 `nsis`
   - resources 覆盖 Live2D、头像、默认模板和许可证文件。
   - CLI 作为 sidecar 或外部二进制随包分发。

4. 补安装器自启动选项。
   - 安装时提供复选框。
   - 应用设置页提供开关。
   - 推荐使用 Tauri autostart plugin 管理运行时开关。

5. WebView2 方案。
   - 内测 ZIP: 启动时检测并给出清晰提示。
   - 正式 NSIS: 使用 bootstrapper/offline installer/fixed runtime 之一。

### P1: 正式公开前完成

- 代码签名。
- 卸载保留/删除用户数据选项。
- 第三方素材授权清单。
- 隐私说明和用户数据删除说明。
- 干净 Windows 10/11 虚拟机安装验证。
- 版本号、构建号、更新日志和回滚策略。

## Tauri/NSIS 配置方向

目标不是立刻写死最终配置，而是按以下方向收敛:

```json
{
  "bundle": {
    "active": true,
    "targets": ["nsis"],
    "resources": [
      "resources/live2d/**/*",
      "resources/licenses/**/*"
    ],
    "windows": {
      "webviewInstallMode": {
        "type": "downloadBootstrapper"
      },
      "nsis": {
        "installMode": "currentUser"
      }
    }
  }
}
```

说明:

- `installMode` 的最终值需要根据是否要避开 UAC 决定。普通用户优先 current-user/per-user 安装。
- 自定义安装路径和自启动复选框可能需要 NSIS template 或 installer hooks 补充。
- 如果使用 Tauri autostart plugin，安装器复选框应写入一个首次启动参数或配置，由应用启动后调用 plugin enable/disable。

## 构建命令建议

内测 ZIP:

```powershell
cd I:\personal-agent
cargo build --release -p conductor-cli
cd apps\desktop
npm run build
cd src-tauri
cargo build --release
```

正式安装包:

```powershell
cd I:\personal-agent\apps\desktop
npm run tauri -- build
```

## 干净环境验证清单

内测 ZIP:

- 解压到 `C:\Temp\PersonalConductorTest`。
- 双击 `bin\conductor-desktop.exe`。
- 不设置环境变量也能启动。
- 删除 `data/` 后再次启动能自动初始化。
- 无 API key 时聊天功能给出配置提示而不是崩溃。
- 设置 API key 后 LLM 连接测试可用。

正式安装包:

- 安装到默认路径。
- 安装到自定义路径。
- 勾选和不勾选开机自启动各测一次。
- 安装后开始菜单、桌面快捷方式可用。
- 重启系统后自启动行为符合选择。
- 卸载后程序目录清理干净。
- 用户数据保留/删除策略符合安装器提示。
