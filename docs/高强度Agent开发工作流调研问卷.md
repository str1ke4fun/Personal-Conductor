# 高强度 Agent 开发工作流调研：给忙不过来的人配个会看摊的小助手

## 开场说明

我们正在做一个给高强度 agent 用户用的桌面小助手。

- 它知道你大概在忙什么，比如当前应用、窗口、项目、agent 会话状态。
- 它能把 Claude Code / Codex / Cursor 这些 agent 的进度和产出收一收，不让它们各说各的。
- 它会维护一份“现在有哪些东西等你看、等你拍板、等你继续推”的清单。
- 它会在合适的时候提醒你：哪个 agent 干完了、哪里需要你看、下一步比较适合先干什么。
- 以后它也可能像 Claude Code 一样，有读写工作区文件、运行命令、调用工具、派发子任务的能力。

## A. 当前 agent 使用痛点

### 1. 同时用多个 agent 的时候，你最容易在哪些地方被绊住？【多选】

- 不知道哪个 agent 已经干完了
- 不知道哪个结果应该先看
- 忘记某个会话还在那边跑
- agent 早就完成了，但我过了很久才发现
- 重新点进去之后，还要先想半天它刚才在干嘛
- 多个终端 / 多个项目之间来回切，脑子需要重新开机
- agent 改了代码，但我不知道重点该 review 哪里
- agent 卡住、等权限、等输入，但我没注意到
- agent 输出太长，读完之前不知道它有没有用
- 没有一个地方能告诉我“现在总共堆了多少事”

- 它觉得我下一步该干什么
- 我大概要花多久才能看完
- 它原来是被我哪个 prompt 派出去的
- 其他：______
