### 变更说明

`summarizer.rs` 从规则模板升级为 LLM 生成摘要。

### 新增接口

```rust

pub struct LlmRequestConfig<'a> {
    pub provider: &'a str,
    pub model: &'a str,
    pub base_url: &'a str,
    pub api_key: Option<&'a str>,

    system: &str,
```

### 摘要格式

```
# <slug> - <iso>

**What**: Claude 修改了 <文件>。

**Where**:
- <绝对路径>

**Why it matters**: <上下文关系>

**What you should check**:
- <检查项 1>
- <检查项 2>
```

- LLM 调用失败时自动回退到规则模板

## 4. 打扰策略优化

### 变更说明

`notify_decider.rs` 实现三级打扰策略和冷却时间控制。

### 新增类型

```rust
#[derive(Clone, Debug, PartialEq, Eq)]
pub enum Urgency {
    Low,      // 桌宠表情切换
    Medium,   // TaskPanel banner
    High,     // 系统通知
}

pub enum Action {
    UpdatePetExpression(PetState),
    EmitToTaskPanel { banner: String, urgency: Urgency },
    SystemNotification { title: String, body: String, urgency: Urgency },
    Silent,
}
```

### 决策树

```
1. 专注模式 / 静默时段 → Silent

2. PendingPileUp
   - oldest > 60min 或 count > 5 → SystemNotification (High)
   - oldest > 30min 或 count > 3 → TaskPanel banner (Medium)
   - 其他 → 仅切桌宠表情

3. UserBackFromIdle
   - idle > 10min + pending > 0 → SystemNotification (High)
   - idle > 5min + pending > 0 → TaskPanel banner (Medium)
   - 其他 → 桌宠点头

4. AgentStalled
   - 首次 → SystemNotification (High)
   - 后续 → TaskPanel banner (Medium)
```

### 冷却时间

- 系统通知：1 小时内最多 1 次
- TaskPanel banner：5 分钟内不重复

### 工作时间检测

- 工作日 9:00-18:00：正常打扰
- 非工作时间：降低打扰强度
- 周末：静默模式

## 13. 2026-05-21 修订：Claude Code 任务触发流

本节覆盖本文档中所有旧的 Claude hook 描述。当前只定义 Claude Code 版本，不引入 Codex 侧任务流。

### 13.1 设计目标

- Hook 全程不得打扰 Claude Code 的正常工作。
- Hook 不应向 Claude Code 注入额外指令、阻塞用户 prompt、要求 Claude 改写输出。
- Hook 只做状态记录、上下文读取、摘要生成和任务列表更新。
- 同一时间可能存在多个 Claude Code 终端，需要能区分“哪个终端/会话正在推进哪个任务”。
- 最终目标是实现并启用 hook 闭环；但在代码侧闭环未验证前，`.claude/settings.json` 保持空 hook，避免未验证实现影响当前 Claude Code 正常使用。

### 13.2 状态模型

任务状态使用以下语义：

| 状态 | 含义 | 触发来源 |
|------|------|----------|
| `in_progress` | Claude Code 已收到用户请求，正在推进该任务 | `UserPromptSubmit` |
| `pending` | Claude Code 已完成一轮产出，等待人审阅 | `Stop` / `SubagentStop` / 权限相关事件 |
| `passed` | 人确认通过 | Conductor UI / CLI |
| `rejected` | 人打回或拒绝 | Conductor UI / CLI |
| `skipped` | 人暂时跳过 | Conductor UI / CLI |

`pending` 在产品文案中显示为“待审”，不是“尚未开始”。

### 13.3 UserPromptSubmit 行为

当 Claude Code 触发 `UserPromptSubmit`：

1. Conductor 读取 hook payload 中的 `cwd`、`session_id`、`transcript_path`、终端/进程标识。
2. Conductor 从用户提交内容和当前工作区提取任务上下文摘要。
3. 如果没有匹配的活跃任务，则创建新任务。
4. 如果存在同一 `session_id` 或终端上下文的活跃任务，则更新该任务摘要。
5. 任务状态设置为 `in_progress`。
6. 任务列表显示 Claude Code 所在终端/会话、当前用户请求摘要、关联工作目录、最近上下文摘要和“正在推进中”的状态。
7. Hook 默认不向 Claude Code 输出 `additionalContext`。只有未来明确开启“非打扰提示”开关时，才允许输出短上下文。

### 13.4 Stop / SubagentStop / 权限事件行为

当 Claude Code 触发 `Stop`、`SubagentStop`，或进入/结束提权请求相关事件时：

1. Conductor 读取同一会话/终端对应的任务。
2. 读取 `transcript_path` 尾部、最近修改文件、权限请求摘要和当前 cwd。
3. 生成“Claude 已完成什么”的上下文摘要。
4. 更新任务列表中的任务内容和 artifact 信息。
5. 任务状态设置为 `pending`，产品文案显示为“待审”。
6. 任务列表显示 Claude 已完成的内容、改动位置或产物路径、需要人检查的重点、所属终端/会话。

如果 Stop 时没有找到对应的 `in_progress` 任务，可以兜底创建一条 `pending` 任务，但这只是兼容路径，不是主流程。

### 13.5 终端/会话归属

每条 Claude Code 任务必须保存会话归属信息：

```rust
pub struct AgentSessionRef {
    pub source: String,          // "claude"
    pub session_id: Option<String>,
    pub terminal_id: Option<String>,
    pub cwd: PathBuf,
    pub started_at: DateTime<Utc>,
    pub last_event_at: DateTime<Utc>,
}
```

匹配优先级：

1. `session_id`
2. `terminal_id`
3. `cwd + 最近活跃时间窗口`

不能只按 cwd 合并任务，因为同一个仓库下可能开多个 Claude Code 终端处理不同任务。

### 13.6 验收标准

- 触发 `UserPromptSubmit` 后，任务列表立即出现一条 `in_progress` 任务。
- 同一 Claude Code 会话继续提交 prompt 时，更新同一任务，而不是重复创建。
- Claude Code 完成一轮输出后，任务变为 `pending`，界面显示“待审”。
- 任务卡片能看出哪个终端/会话在做什么。
- Hook 不改变 Claude Code 的正常 prompt 内容，不弹窗，不阻塞，不要求用户额外确认。
- 任务面板已打开时能收到任务创建/更新，或以等价机制保证用户不需要手动刷新才能看到最新状态。
- 权限请求相关事件和 `SubagentStop` 至少有明确接入路径；如果 Claude Code 当前 hook 能力不足，必须在实现说明中标注降级行为。
- `.claude/settings.json` 只有在上述闭环通过验证后才写入 hook；写入必须由用户显式选择。当前空 hook 是过渡保护，不是最终目标状态。

### 13.7 启用闸门与当前代码差距

在把 hook 写入 `.claude/settings.json` 前，必须先补齐并验证：

1. `UserPromptSubmit` 创建/更新任务后，TaskPanel 能自动展示新任务或收到刷新事件。
2. `UserPromptSubmit` 的任务摘要必须来自用户请求、cwd、transcript tail 或已有上下文摘要，不能把 `summary_ref` 当成内联文本和文件路径混用。
3. `Stop` / `SubagentStop` / 权限请求相关事件必须按 `session_id -> terminal_id -> cwd + 时间窗口` 匹配同一任务，不能只取第一个 `source=claude && in_progress` 任务。
4. Stop 阶段的文件扫描和 LLM 摘要不能同步拖慢 Claude Code；重型工作必须异步或快速失败。
5. `.claude/settings.local.json` 中的命令 permission 只表示允许执行相关命令，不等于启用 hook；真正启用以 `.claude/settings.json` 的 hook 配置为准。
