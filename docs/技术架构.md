## 2. 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 桌面壳 | Tauri v2 | Rust 后端 + WebView 前端 |
| 前端 | React 18 + TypeScript + Vite | 窗口渲染、设置面板、聊天 |
| 动画 | pixi.js + Live2D Cubism SDK | 桌宠角色渲染 |
| 后端逻辑 | Rust (conductor-core) | 所有业务逻辑 |
| 持久化 | SQLite (sqlx) | 任务、聊天、情绪、记忆 |
| 向量嵌入 | fastembed (BGE-small-en-v1.5) | 记忆语义搜索 |
| LLM | OpenAI 兼容 / Anthropic 兼容 | 对话、摘要、智能监控 |
| 感知层 | conductor-sense | OS 级窗口焦点/空闲检测 |

## 3. 核心模块

### 3.1 情绪系统 (expression.rs)

基于 PAD 模型简化版，连续的 valence × arousal 映射到 7 个离散 MoodZone:

| MoodZone | 条件 | 中文标签 |
|---|---|---|
| Happy | v>0.3, a>0.3 | 开心 |
| Content | v>0.3, a≤0.3 | 满足放松 |
| Frustrated | v<-0.3, a>0.5 | 懊恼 |
| Sad | v<-0.3, a≤0.5 | 低落 |
| Shy | v≤-0.1, a>0.4 | 害羞 |
| Bored | -0.3≤v≤0.3, a>0.5 | 无聊 |
| Neutral | 其他 | 平静 |

- 每 60 秒衰减: valence→0.0 (±0.02), arousal→0.15 (±0.01)
- 事件触发: on_tool_success, on_tool_failure, on_user_message, on_llm_error 等
- 持久化: mood_state 表 (SQLite)

### 3.2 好感度系统 (affection.rs)

5 个关系阶段，基于好感度值 (0-100) + 迟滞机制:

| 阶段 | 值范围 | 迟滞升级阈值 | 迟滞降级阈值 |
|---|---|---|---|
| Stranger | 0-19 | >22 | ≤17 |
| Acquaintance | 20-39 | >42 | ≤37 |
| Colleague | 40-59 | >62 | ≤57 |
| Friend | 60-79 | >82 | ≤77 |
| CloseFriend | 80-100 | - | - |

- 交互类型: Chat(+1), ToolSuccess(+1), TaskApproved(+2), TaskRejected(-2) 等
- 每日衰减: 3+ 天无交互 → -1/天
- 阶段保护: 不会跌破当前阶段最低值 - 5

### 3.3 智能监控 (smart_monitor.rs)

Pacer → Smart Monitor → 用户通知 管道:

```
Pacer (每5分钟)
  ├─ PendingPileUp: ≥5 待处理 或 最老任务 >60 分钟
  ├─ AgentStalled: 进行中任务 >30 分钟无进展
  └─ NoActivityHour: 每小时整点
       ↓
Smart Monitor (LLM 决策)
  ├─ 收集上下文: 任务列表、心情、好感度、最近对话
  ├─ 调用 LLM 判断: 是否通知？消息内容？紧急程度？
  └─ 回退: LLM 不可用时使用硬编码默认值
       ↓
用户通知
  ├─ pet_state 更新
  ├─ taskpanel_banner (低/中/高紧急)
  └─ 系统通知 (中/高紧急)
```

### 3.4 人格系统 (persona.rs)

Persona 包含:
- 人格特质 (PersonalityTrait: name, value, description)
- 语气风格 (tone)
- Prompt 模板 (可变量替换)
- 图像生成 prompt

`build_system_prompt` 在每次对话时注入人格特质到 LLM 系统提示。

### 3.5 记忆系统 (memory.rs)

- 向量嵌入: fastembed BGE-small-en-v1.5 (384 维)
- 回退: fastembed 不可用时使用 hash-based 伪嵌入
- 语义搜索: cosine similarity, 阈值 0.3
- 分类存储: memory_chunks 表 (SQLite)

## 4. 数据流

### 4.1 用户对话

```
用户输入 → ChatPanel/PetWindow
  → invoke('send_chat_message')
  → chat::send()
  → build_system_prompt (注入 mood + persona + memory + tasks)
  → llm::call_with_tools_with_messages (支持工具调用循环)
  → 更新 mood/affection
  → 返回 ChatReply
  → 前端显示
```

### 4.2 主动提醒

```
Pacer (5分钟轮询) → PacerAlert
  → Smart Monitor::evaluate (LLM 决策)
  → SmartDecision { notify, message, urgency, pet_state }
  → emit('pet_state') + emit('taskpanel_banner')
  → 系统通知 (中/高紧急)
```

### 4.3 焦点感知

```
conductor-sense focus_watcher
  → FocusEvent { process_name, title, is_maximized }
  → initiative::update_initiative_context
  → maybe_emit_proactive_prompt (LLM 生成问候)
  → emit('pet_message')
```

## 5. Hook 接入

Claude Code hooks 通过 `conductor-cli` 转发到 `conductor-core`:

| Hook | 事件 | 处理 |
|---|---|---|
| Stop | Claude Code 会话结束 | 生成摘要，创建/更新任务 |
| SubagentStop | 子代理结束 | 同 Stop |
| UserPromptSubmit | 用户提交 prompt | 记录 in_progress 任务 |
| PreToolUse | 工具调用前 | 记录权限上下文 |
| PostToolUse | 工具调用后 | 记录工具结果 |
| Notification | 通知事件 | 记录事件 |

## 7. 事件系统

Tauri 事件 (前端监听):

| 事件 | 数据 | 说明 |
|---|---|---|
| `pet_state` | string (idle/working/update/quiet/new_task) | 桌宠状态 |
| `pet_expression` | { avatar_id, activity_variant, mood_zone, relationship_stage } | 表情状态 |
| `pet_message` | { id, kind, content, action } | 主动消息 |
| `taskpanel_banner` | { banner, urgency } | 任务面板横幅 |
| `tasks_changed` | () | 任务列表变更 |

## 8. 配置

`CoreConfig` 结构 (config.rs):

```rust
pub struct CoreConfig {
    pub llm: LlmConfig,           // LLM provider/model/key/allowed_tools
    pub reminders: ReminderConfig, // 工作时间、安静模式
    pub pet: PetConfig,            // 头像锁定等
    pub persona: PersonaConfig,    // 名称、风格、技能
    pub proactive: ProactiveConfig,// 主动对话开关、冷却时间
    pub codex: CodexConfig,        // Codex 集成
    pub feishu: Option<FeishuConfig>, // 飞书 webhook
}
```
